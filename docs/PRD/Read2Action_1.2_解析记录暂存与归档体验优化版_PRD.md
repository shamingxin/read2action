# Read2Action 1.2 PRD

**版本名**：Read2Action 1.2「解析记录暂存与归档体验优化版」  
**文档角色**：记录 1.2 版本的产品定义，待立项后启动开发。  
**状态**：📋 PRD 已定义，尚未立项开发。开发前需先生成 1.2 版本启动方案。

---

## 1. 版本定位

在 1.1 已有真实 AI 解析基础上，**补全本地解析记录的暂存、归档、找回与空状态体验**，让产品更像一个「不会丢内容」的真实工具。

**核心三原则**：
1. **解析后不丢**：从任何入口解析后，结果自动暂存，用户无需担心丢失。
2. **没解析过时不假装有内容**：清除假数据，首次进入空状态真实呈现。
3. **项目内解析能自动归档**：从项目页发起解析，自动归属当前项目。

**不接账号、不接数据库**，所有记录仍保存在当前浏览器本地（localStorage）。

---

## 2. 版本目标

1. 统一真实空状态，去除所有默认 mock 假数据展示。
2. 全局解析结果自动暂存为「未归档」记录，出现在最近列表。
3. 项目内解析结果自动归档到当前项目。
4. 最近列表和结果页支持「未归档 / 已保存」状态区分。
5. 修复影响主链路的交互 bug（保存状态、Toast 文案、列表异常、空状态缺失等）。
6. 明确本地保存口径，告知用户数据范围。

---

## 3. 用户价值

| 场景 | 1.1 体验 | 1.2 目标体验 |
|------|----------|--------------|
| 首次进入，没有内容 | 看到 mock 假数据 | 看到真实空状态，引导开始使用 |
| 首页解析后没有保存 | 结果消失，找不回来 | 自动暂存到「最近」，可后续归档 |
| 想知道之前解析了什么 | 只能看 mock 笔记 | 最近列表展示真实记录，含状态标签 |
| 在项目页发起解析 | 需要手动选项目保存 | 自动归档到当前项目 |
| 想把暂存记录保存到项目 | 不支持 | 可从最近列表或详情页保存到项目 |
| 担心换设备数据丢失 | 没有说明 | 明确提示「仅在当前浏览器」 |

---

## 4. 范围内

1. **统一切到真实空状态**：首次进入时，最近列表、项目列表、项目内笔记列表，均不展示 mock 假数据，展示空状态引导。
2. **全局解析结果自动暂存**：从首页 / 侧栏「新笔记」入口解析后，自动生成 `savedStatus: "temporary"` 的未归档记录，出现在最近列表。
3. **未归档记录可后续保存到项目**：从最近列表或详情页，可将「未归档」记录保存到指定项目，状态变为 `savedStatus: "saved"`。
4. **项目内解析结果自动归档**：从项目页顶部解析条发起解析后，自动保存到当前 `projectId`，`savedStatus: "saved"`。
5. **最近列表支持状态区分**：展示「未归档」/ 「已保存」标签，并可从最近列表直接归档。
6. **结果页根据来源展示不同状态**：全局解析显示「已自动暂存」，项目内解析显示「已保存到项目」。
7. **明确本地保存口径**：在结果页或设置处展示说明：「当前版本记录保存在当前浏览器本地，清除浏览器数据或更换设备后不会同步。」
8. **修复影响主链路的交互 bug**（详见「Bug 修复范围」）：
   - 保存状态不清（Toast 显示后按钮状态未变）
   - 按钮无反馈（点击无 loading / disabled 状态）
   - 最近列表状态异常（刷新后 mock 混入真实记录）
   - 详情页打不开本地记录（路由未命中本地记录时 notFound）
   - 刷新后状态异常（hydration 导致的短暂 mock 态）
   - Toast 文案不准确（来源不同时文案应有区分）
   - 空状态缺失（没有数据时无引导）

---

## 5. 范围外

1. 不接数据库（仍为 localStorage，明确边界）。
2. 不做登录注册。
3. 不做多端同步。
4. 不做文件上传解析。
5. 不做链接抓取（URL 仍仅提示复制正文）。
6. 不做多模型 Provider 架构（单 DeepSeek）。
7. 不做大视觉改版，不推翻六页结构。
8. 不做方法论自动生成。
9. 不重写 `/api/analyze`。
10. 不破坏 `/parsing` 防重复请求机制（A+B+C 继续保留）。

## 6. 核心流程

### 6.1 全局解析流程（来源：首页 / 侧栏新笔记）

```text
首页 / 侧栏「新笔记」输入文本，点「开始解析」
→ /parsing → POST /api/analyze（防重复）
→ 成功 → 自动写入 localStorage（savedStatus: "temporary", sourceContext: "global"）
→ /result 展示：「已自动暂存」状态提示
→ 可选：点「保存到项目」→ 选择项目 → savedStatus 变为 "saved"，projectId 写入
→ 最近列表：展示该记录（标记「未归档」或「已保存」）
```

### 6.2 项目内解析流程（来源：项目页顶部解析条）

```text
进入 /projects/[projectId]，在顶部解析条输入文本，点解析
→ /parsing → POST /api/analyze（携带 projectId，防重复）
→ 成功 → 自动写入 localStorage（savedStatus: "saved", sourceContext: "project", projectId）
→ /result 展示：「已保存到项目」状态提示
→ 项目页笔记列表即时出现新记录
→ 最近列表：展示该记录（标记「已保存」）
```

### 6.3 空状态流程

```text
首次进入（无 localStorage 数据）
→ 最近列表：显示空状态组件（「还没有解析记录，从首页开始」）
→ 项目列表：显示真实项目（如有预设），笔记列表为空时展示空状态
→ 不展示 mock 假笔记
```

## 7. 数据 / 状态规则

### 数据模型扩展

在现有 `Note` 类型基础上新增字段：

```ts
// 解析来源
sourceContext: "global" | "project"

// 保存状态
savedStatus: "temporary" | "saved"

// 关联项目（全局解析未归档时可为空）
projectId?: string

// 时间戳
createdAt: string
updatedAt: string
```

### 关键状态规则

| savedStatus | sourceContext | projectId | 含义 |
|-------------|---------------|-----------|------|
| "temporary" | "global" | undefined | 全局解析，未归档 |
| "saved" | "global" | "sha" 等 | 全局解析，已保存到项目 |
| "saved" | "project" | "sha" 等 | 项目内解析，自动归档 |

### 最近列表规则

- 展示所有 `savedStatus: "temporary"` 和最近 `savedStatus: "saved"` 的记录。
- 按 `updatedAt` 降序，上限条数（1.2 具体定，建议 10 条）。
- 首次进入无 localStorage 数据时，展示空状态，不展示 mock。

### Mock 数据策略

- 1.2 起，最近列表、项目内笔记列表默认不展示 mock 假数据。
- mock 数据保留在代码中（作为开发/演示参考），但不默认注入 UI。
- 开发期可通过开关（如 `?mock=true` 参数）临时启用 mock 视图。

## 8. 验收标准

| 项目 | 标准 |
|------|------|
| 无 mock 污染 | 首次进入（清除 localStorage 后），最近列表、项目笔记列表均为空状态，无假数据 |
| 自动暂存 | 首页解析后，不点保存也能在最近列表找到该条记录 |
| 刷新不丢 | 刷新后，暂存记录和已保存记录均存在 |
| 状态归档 | 未归档记录可通过「保存到项目」变为已保存 |
| 项目内自动归档 | 从项目页发起解析后，记录自动出现在当前项目列表 |
| 状态区分 | 最近列表能区分「未归档」和「已保存」 |
| 详情页可访问 | 本地保存的记录（含暂存）可正常打开详情页 |
| 单次解析 | Network 仍仅 1 条 POST /api/analyze（防重复机制未破坏） |
| API Key 安全 | API Key 仍只在服务端读取 |
| 工程质量 | npm run test / lint / build 通过 |
| 线上验收 | 线上部署后重新验收，以上条目均通过 |

## 9. 现实校准

- **localStorage 仍是明确边界**：1.2 不引入数据库，本地保存口径须向用户明确说明。
- **不是全量重构**：1.2 以最小改动补全「记录不丢」体验，不大改架构，不重写已有稳定链路。
- **不破坏 A+B+C 防重复机制**：解析链路的防重复方案不动，只在外层增加 sourceContext 传递。
- **mock 数据保留但不默认展示**：保留 mock 代码用于开发参考，但产品层面切到真实空状态。
- **1.3 才做账号**：1.2 结束后，数据仍在 localStorage，跨设备同步列为 1.3 目标。
