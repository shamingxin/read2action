# Read2Action 1.0 PRD

**版本名**：Read2Action 1.0「MVP 演示版」  
**文档角色**：记录 1.0 版本的产品定义，供版本回溯与接力参考。  
**状态**：✅ 已上线冻结（2026-05-14 前后）。不再扩展功能，不做视觉大改。

---

## 1. 版本定位

Read2Action 1.0 是一个 **AI 知识整理与行动转化工具的 MVP 演示原型**。

核心命题：帮助用户将文本、笔记或想法快速转化为结构化笔记，并保存到项目中，方便后续查看、复用和沉淀。

**1.0 的角色定位**：不是生产工具，而是一个视觉接近真实 SaaS、主流程可演示、代码结构清晰、可用于作品集展示的高保真原型。

---

## 2. 版本目标

1. 实现 6 个核心页面，主流程完整可跑通。
2. 视觉还原 Figma 设计稿，接近真实 SaaS 产品风格。
3. 代码结构清晰，具备持续迭代能力。
4. 完成 GitHub + Vercel 部署，线上可访问。
5. 为后续 1.1（真实 AI 接入）打下代码基础。

---

## 3. 用户价值

| 用户动作 | 产品响应 | 感知价值 |
|----------|----------|----------|
| 粘贴一段文本或想法 | 进入解析中页，四步进度展示 | 感受到 AI 处理过程 |
| 等待解析完成 | 进入结果页，结构化总结展示 | 内容被整理成可行动形式 |
| 点击保存到项目 | 写入项目笔记列表 | 内容被沉淀，可后续复用 |
| 进入项目 / 详情页 | 查看笔记列表与详情 | 知识被归档，随时可查 |

---

## 4. 范围内

1. 6 个核心页面：首页、AI 解析中、解析结果、项目目录、笔记详情（内容总结 Tab）、笔记详情（原文对照 Tab）。
2. 左侧固定 Sidebar，全局复用。
3. 主流程可跑通：首页输入 → `/parsing` → `/result` → 保存到项目 → `/projects/[id]` → 笔记详情。
4. Mock 数据驱动：项目列表、笔记列表、解析结果均为静态 mock。
5. 本地 / 前端状态：保存到项目写入前端状态，刷新后恢复 mock。
6. 基础交互状态：空输入置灰 Toast、解析中取消、行动清单 checkbox 本地切换、Toast 占位等。
7. 技术栈：Next.js App Router + React + TypeScript + Tailwind CSS + shadcn/ui + lucide-react。
8. Vercel 生产部署，已上线可访问。

## 6. 核心流程

```text
首页：粘贴文本，点「开始解析」
→ /parsing：四步进度依次展示（读取内容 → 提炼要点 → 结构化处理 → 生成结果），约 3.5～4.5s
→ /result：结构化总结展示（一句话总结、核心观点、行动清单、知识卡片）
→ 保存到项目弹窗：选择项目，默认「沙」，点保存
→ Toast「已保存到项目」，可跳转 /projects/sha
→ /projects/[projectId]：项目笔记列表，可搜索
→ /projects/[projectId]/notes/[id]：笔记详情，内容总结 / 原文对照 Tab 切换
```

---

## 7. 数据 / 状态规则

| 数据层 | 说明 |
|--------|------|
| 项目列表 | 静态 mock（`projects.mock.ts`），含「沙」「项目一」等预设项目 |
| 笔记列表 | 静态 mock（`notes.mock.ts`），含多条示例笔记 |
| 解析结果 | 静态 mock（`result.mock.ts`），固定展示内容 |
| 保存到项目 | 写入前端 state，刷新后恢复 mock 初态；1.0 明确不持久化 |
| 解析中进度 | 前端定时器驱动，四步依次切换 |

**关键规则**：
- `/parsing` 取消解析：清除定时器，回首页，无幽灵跳转。
- 空输入开始解析：Toast「请输入内容后再解析」，不跳转。
- Mock 数据与前端状态并存属 1.0 预期，非 bug。

## 9. 现实校准

- **1.0 不是生产工具**：mock 数据、前端状态，刷新后不持久，属版本预期。
- **不追求像素级还原**：字体、间距与 Figma 细差可接受，主结构与视觉风格一致即可。
- **不扩展 1.0 范围**：1.0 已冻结，发现缺失功能应立项到 1.1 或 1.2，不在 1.0 补。
- **代码可迭代是关键**：结构清晰，组件化，便于 1.1 接入真实 AI 时最小改动。
