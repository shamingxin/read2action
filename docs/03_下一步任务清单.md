## Read2Action 1.1 范围边界（真实 AI 文本总结版）

> 与 `02_当前进度快照.md` **§8**、`04_重要决策日志.md` **2026-05-16** 条目一致；立项与排期须遵守。

### 定义

- **Read2Action 1.1 = 真实 AI 文本总结版**：在既有 **六页主流程** 下，将核心解析结果从 **mock** 升级为 **真实大模型基于用户输入文本生成的结构化总结**，并在 **结果页** 展示 **真实输出**。

### 1.1 只做

1. 用户输入 **一段待解析文本**（**1.1 以纯文本为可靠输入口径**；链接形态见下条「轻提示」策略）。  
2. **调用 AI**（后端 API + 模型）生成 **可映射到现有 UI 的结构化总结**。  
3. **结果页展示真实 AI 结果**（替换 mock 的核心体验；保存到项目等链路在立项中明确是否仍走当前前端状态或最小扩展，但 **不借机引入数据库/账号**）。

### 1.1 暂不做

- **链接全文抓取**、自动抓取网页/文章正文、任意 URL 内容拉取。  
- **登录注册**、**多用户**、账号体系与权限。  
- **数据库** 与服务端持久化（项目/笔记/用户数据落库等）。  
- **文件上传** 及基于文件的解析。  
- **浏览器插件**、RSS/剪藏类站外自动采集。  
- **复杂知识库**（超出「单段文本 → 单次结构化总结」的架构）。  
- **方法论自动生成**（不在 1.1 内把「方法论沉淀」升级为 AI 自动生成能力）。

### 用户输入为链接时（1.1）

- **不实现**自动读取链接正文。  
- **仅做轻提示**，建议文案口径：**「当前版本暂不支持自动读取链接正文，请复制正文后粘贴。」**（实现形态：Toast / 输入区提示等，立项时定稿。）

### 边界目的

- **避免 1.1 范围失控**（抓取、账号、落库最易把 1.1 拖成小 2.0）。  
- **先把产品从 mock 演示变成真实可用**：优先跑通 **「文本进 → 真 AI 出 → 结果页真展示」**。

## 1. 1.1 准备期 — 建议动作（文档 / 方案为主）

1. **接口形态**：对齐 **`/api/analyze`**（或等价路径）的请求/响应 JSON schema；**请求体以纯文本为主**；错误码与超时策略。  
2. **模型与密钥**：选型（单一供应商优先）、环境变量与 Vercel Secret、成本与限流。

5. **链接形态与轻提示**：如何检测「主要为 URL 的输入」、提示文案与不打断主流程的策略（**不**在 1.1 内做抓取兜底）。  
6. **安全与合规**：用户输入日志是否存储、PII、内容审核策略（初版可「不落库」）。  
7. **验收标准**：单独立项文档中写清「1.1 Done」的 E2E 清单（含线上 staging），且 **验收范围不得超过** 上文「只做 / 暂不做」。

## 7. Cursor 开发分阶段指令（历史参考 — 1.0 已基本完成）

> 以下段落来自原《发给 Cursor 材料清单》，适用于 **从零重做教学** 或 **新成员模拟路径**；当前仓库 **1.0 已完成**，请勿在未立项下机械执行「继续开发」类指令。

### 第一阶段：项目初始化 + 基础结构

```text
请先完成第一阶段：
1. 初始化 Next.js + TypeScript + Tailwind CSS 项目
2. 配置 shadcn/ui 和 lucide-react
3. 创建基础目录结构
4. 创建 types/index.ts
5. 创建 data/projects.mock.ts、data/notes.mock.ts、data/result.mock.ts
6. 创建 AppLayout 和 AppSidebar
7. 先保证首页能显示左侧 Sidebar 和右侧空内容区

不要开发全部页面，完成后告诉我改了哪些文件。
```

### 第二阶段：首页

```text
请开发首页 / 新笔记输入页：
1. 还原 Figma 首页布局
2. 实现统一大输入框
3. 输入为空时开始解析按钮禁用或置灰并可 Toast
4. 输入有内容时按钮可点击
5. 点击开始解析跳转 /parsing
6. 模型下拉可以先做静态
7. 底部展示四个“你将获得”的卡片
```

### 第三阶段：解析中

```text
请开发 AI 解析中页面：
1. 路由为 /parsing
2. 展示标题与四步进度
3. 展示 LIVE PREVIEW 骨架屏
4. 四步依次展示完成后跳转 /result（时长以产品为准）
5. 点击取消解析返回首页并清除定时器，无幽灵跳转
```

> **注**：早期材料写「2–3 秒」「确认弹窗」；**9.3 终态**为四步完整展示 + 约 3.5～4.5s 量级、取消直接回首页 — 以 **`01` / `02`** 与代码为准。

### 第四阶段：解析结果

```text
请开发解析结果页：
1. 路由为 /result
2. 使用 result.mock.ts 数据
3. 展示一句话总结、核心观点、行动清单、知识卡片
4. 实现保存到项目弹窗，默认保存到“沙”
5. 保存成功后 Toast 提示“已保存到项目”
6. 可跳转 /projects/sha
7. 编辑和导出先用 Toast mock
```

### 第五阶段：项目目录

```text
请开发项目目录页：
1. 路由为 /projects/[projectId]
2. 展示当前项目名
3. 展示顶部快速解析输入条
4. 展示项目内笔记列表表格
5. 点击笔记行进入详情页
6. 搜索支持按标题和标签前端过滤
7. 更多操作和分页可以先静态或 mock
```

### 第六阶段：笔记详情

```text
请开发笔记详情页：
1. 路由为 /projects/[projectId]/notes/[id]
2. 展示面包屑、标题、来源、标签、创建时间、字数
3. 实现内容总结 / 原文对照 Tab 切换
4. 内容总结展示总结、知识卡片、核心观点、行动清单、原文摘录
5. 原文对照展示原始内容
6. checkbox 支持本地切换状态
```

### 第七阶段：统一视觉与交互

```text
请统一检查所有页面：
1. 是否接近 Figma
2. Sidebar 是否复用
3. 页面间距、圆角、边框、阴影是否统一
4. hover / focus / disabled 状态是否存在
5. 文案是否正确
6. 是否没有出现“保存到知识库”等旧文案
7. 主流程是否能完整跑通
```
