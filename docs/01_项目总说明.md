## 3. 产品定位

Read2Action 是一个 **AI 知识整理与行动转化** 工具。

**一句话**：帮助用户将文本、链接、笔记或个人想法快速转化为结构化笔记，并保存到项目中，方便后续查看、复用和沉淀。

**价值链**：

```text
收藏内容 → 理解内容 → 提炼行动 → 沉淀知识 → 后续复用
```

## 4. 1.0 核心主流程

```text
首页输入内容
→ 点击开始解析
→ 进入 AI 解析中页面（四步完整展示后进入结果）
→ 进入解析结果页
→ 点击保存到项目
→ 保存成功 Toast
→ 在项目「沙」中出现新笔记
→ 点击笔记进入详情页
→ 在内容总结 / 原文对照 Tab 之间切换
```

1.0 阶段使用 mock 数据与本地状态跑通；不接真实后端、不接真实 AI API（**1.1** 再接入，见下文版本路线）。

## 5. 1.0 页面范围与路由

**必须实现的 6 个核心页面**：

1. 首页 / 新笔记输入页  
2. AI 解析中页  
3. 解析结果页  
4. 项目目录 / 项目内笔记列表页  
5. 笔记详情页：内容总结 Tab  
6. 笔记详情页：原文对照 Tab  

**布局**：左侧固定侧边栏 + 右侧主内容区。

**推荐路由**：

```text
/                                首页 / 新笔记输入页
/parsing                         AI 解析中页
/result                          解析结果页
/projects/[projectId]            项目目录 / 项目内笔记列表页
/projects/[projectId]/notes/[id] 笔记详情页
```

MVP 可使用 localStorage / mock data / query 简化数据传递。

## 12. 交互状态要求（摘要）

### 首页

- 输入为空：开始解析按钮置灰（`aria-disabled` 等）仍可点击并 Toast「请输入内容后再解析」（**9.1 终态**）。  
- 输入有内容：可跳转 `/parsing`。  
- 左侧「新笔记」：回首页并清空输入。  
- 模型选择：Dropdown 风格，选项如 Auto / GPT-5.5 / Sonnet 4.6（静态）。

### 解析中页 `/parsing`

- 进入后显示四步进度；**9.3 终态**：四步依次展示（读取内容 → 提炼要点 → 结构化处理 → 生成结果）后再跳转 `/result`；总时长约 3.5～4.5s 量级（以代码为准）。  
- 取消解析：回 `/` 并清除定时器，**无幽灵跳转**。（交接文档曾列「确认弹窗」为可选细修，见 `03_下一步任务清单.md`。）

### 解析结果页 `/result`

- 展示一句话总结、核心观点、行动清单、知识卡片；「原文对照」Tab 可为占位。  
- 保存到项目：弹窗，默认「沙」；确认后 Toast「已保存到项目」并可跳转 `/projects/sha`。  
- 编辑 / 导出 / 更多：Toast 或 Dropdown 占位。

### 项目目录页

- 展示项目名；顶部快速解析条；行进入详情；搜索前端过滤；更多与分页可 mock。

### 笔记详情页

- 默认内容总结 Tab；原文对照 Tab 按 Figma「06-详情里的原文」实现（历史 nodeId：`100:2139`，以设计文件为准）。  
- 面包屑可回项目列表；行动清单 checkbox 本地切换。

**更细的跳转表**见 `01` 附录或原逻辑全文已并入 **`05_Cursor执行规则.md` 附录「交互与跳转逻辑（完整表）」** — 为控制篇幅，完整表格放在 `05` 末尾。

**关键词**：轻量 / 克制 / 清晰 / 理性 / 高级感 / AI 辅助感 / 知识工具感 / 真实 SaaS / 白底 + 浅灰背景 + 紫蓝色点缀。

## 17. 产品版本路线（1.0 / 1.1 / 2.0）

### 17.1 1.0（已完成并上线）

- mock + 本地状态，6 页主流程与占位交互稳定；已 GitHub + Vercel 闭环。  
- **不接**真实 AI API、数据库、登录 — **1.0 范围内始终成立**；**已冻结**。

### 17.2 1.1（后续计划 — 当前默认「下一阶段」）

- **目标**：用户输入经 **后端 API** 调用 **真实大模型**，生成 **结构化笔记**，**替换** mock 解析结果。  
- **优先范围**：新增 `/api/analyze`（或等价路径）；接入一个真实 AI API；请求体携带用户输入；响应映射到现有 UI（如 `ParseResultPreview`）；**保留 `/parsing` 四步加载体验**（与真实进度衔接实现时另定）。  
- **仍不做（与规划一致）**：登录、数据库、真实删除、真实导出。  
- **准备期**：未立项前**不默认改业务代码**（见 `03`、`04`）。

### 17.3 2.0（再考虑）

数据库持久化项目与笔记；登录注册；多项目真实管理；历史记录；搜索；导出；多端同步等。
