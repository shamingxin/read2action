# Read2Action · 文档目录说明

本目录按 **三类** 组织，便于新对话 `@` 引用与版本接力：

| 目录 | 用途 |
|------|------|
| **`核心记录/`** | 长期维护的 **`01`～`07`** 七份中文核心文档，外加 **`Read2Action_协作流程SOP_v1.0.md`**（协作流程 SOP）。 |
| **`版本启动方案/`** | 各版本 **开工前** 的启动方案、契约、小 PRD（当前含 **1.1** 的 **`08_1.1真实AI文本总结启动方案.md`**；后续 **1.2** 等可并列新增）。 |
| **`阶段复盘/`** | 各阶段 / 版本的 **复盘与验收事实归档**（如 1.0 一页式总结、**9.3 走查全文**、**1.1 P0 排障实录**、开发阶段分表、1.1 上线收尾、1.1 正式总复盘等）。 |

## docs 整理原则（`01`～`07` vs 阶段复盘 vs 版本启动方案）

1. **`核心记录/01`～`07`**：**保持精简**，服务 **新对话接力** 与 **项目推进**；只写 **当前状态、关键决策、下一步、边界与索引**；**不写**可独立成篇的长过程表。  
2. **`阶段复盘/`**：允许 **更详细**；必须保留 **过程、问题、尝试路径、最终方案、工作流反思、作品集素材**；**禁止**为迁就 `02` 篇幅而 **过度压缩成一句结论**（可写摘要 + **链到本目录实录**）。  
3. **`版本启动方案/`**：保持 **结构化**；重点写 **版本目标、范围、不做什么、功能边界、执行步骤、验收标准**（与 `08` 体例一致）。

**默认接力**：优先 **`@docs/核心记录/02_当前进度快照.md`**，并配合 **`docs/核心记录/03_下一步任务清单.md`**、**`docs/核心记录/04_重要决策日志.md`**、**`docs/核心记录/05_Cursor执行规则.md`**。进入 **1.x 代码或契约细节** 时补充 **`docs/版本启动方案/`** 下 **`08`** 或未来 **`09`** 等文件。

**路径约定**：下文与索引中的 `docs/...` 路径均自 **仓库根目录** 起算。
