## 1. 1.2 版本目标

Read2Action 1.2 在 **1.1 真实 AI 文本总结版** 已冻结的基础上，**在不接登录、不接数据库** 的前提下，补全 **「解析内容不会丢」** 的本地暂存与归档体验。

核心目标：

> 用户解析出的内容，不能只停留在一次性结果页里，而应该可以被暂存、找回、保存到项目，并形成基础记录生命周期。

本阶段 **不** 借机引入数据库、登录、链接抓取或多模型 Provider；重点区分 **全局未归档（temporary）** 与 **项目内已归档（saved）** 两条路径，让用户「解析过就能找回」。

## 3. 本阶段关键问题与解决过程

| 问题 | 现象 / 本质 | 解决过程摘要 |
|------|-------------|--------------|
| **未保存内容找回链路不完整** | 1.1 解析结果易成一次性内容，用户无安全感 | 1.2-B 建立全局 **temporary** 自动暂存 + 「最近」列表找回 |
| **保存到项目在返回结果页后无反应** | 从「最近」进入未归档详情页后，「保存到项目」按钮失效 | 1.2-D 修复详情页保存链路，与 `/result` 行为对齐 |
| **项目内解析归档关系需补全** | 项目页发起解析后仍需手动保存 | 1.2-C 项目内解析成功后自动 **saved + projectId** |
| **原文对照展示不完整或非真实输入** | 占位内容削弱 AI 结果可信度 | 1.2-E 贯通 `rawContent` 自 API → session → localStorage → 各页 |
| **未归档记录面包屑点击导致 404** | 未归档内容无项目归属，面包屑制造错误入口 | 1.2-E 未归档详情页 **隐藏整个面包屑**；已归档保留 |
| **`/parsing` 时间从 17s 闪回 20s** | 固定倒计时 + StrictMode 导致状态不稳定 | 1.2-E 改为 **真实已用时间** + ref 防闪回；不做 AI 耗时预测 |
| **宽屏内容区域过宽** | 大屏下阅读宽度失控，影响专业感 | 总回归 P2 `r2aContentPageShell`（`max-w-[960px]`） |
| **Cursor 长任务容易卡顿** | 大任务导致上下文混乱、中途停住 | 拆成 1.2-A～E 小步；每轮明确范围 / 禁止项 / 验收标准；长对话新开 agent |

## 4. 最终解决方案

1. **用本地暂存机制补全最近记录** — `localStorage`（`r2a:localSavedNotes`）承担 MVP 阶段记录生命周期；解析成功即 **temporary** 写入，刷新不丢。
2. **区分未归档记录与项目内归档记录** — **temporary**（全局未归档）vs **saved + projectId**（项目内已归档）；保存操作将 **temporary → saved**。
3. **明确结果页、详情页、项目详情页之间的记录生命周期** — 输入 → 解析中 → 生成结果 → 自动暂存 → 最近找回 → 保存到项目 → 项目内管理。
4. **保持 `/api/analyze` 单次请求机制不被破坏** — 保留 1.1 **A+B+C** 防重复；本地 dev 仍建议 **`next dev --webpack`**。
5. **使用阶段性验收和小任务推进，避免一次性大改** — 每小步列文件清单 → 用户确认 → 实现 → 本地验收 → commit → 下一小步。

## 5. 设计师 / 验收者视角复盘

### 5.1 版本本质

1.2 **不是大功能版本**，而是 **「记录生命周期」补全** — 从「能生成」走向「能管理」。

### 5.2 验收路径清单（固定）

验收时必须覆盖以下路径，不能只验主路径：

1. **新解析** — 首页 / 项目页发起解析  
2. **返回结果页** — `/result` 展示与保存  
3. **最近找回** — 侧栏「最近」进入未归档详情  
4. **保存项目** — `/result` 与未归档详情页均可保存  
5. **项目内解析** — 自动归档到当前项目  
6. **详情页查看** — 已归档 / 未归档详情页内容正确  
7. **刷新** — 整页刷新后数据与入口仍正确  
8. **宽屏 / 小屏** — 大屏阅读宽度收敛；小屏响应式正常  

### 5.3 体验设计侧后续需提前定义

- **记录状态**（temporary / saved / 归档关系）  
- **入口来源**（首页 / 项目 / 最近 / 直链）  
- **保存关系**（升级路径、跳转目标）  
- **异常状态**（无 session、无效 URL、空状态）

### 9.2 1.4「文件上传 / 更多输入来源版」

**定位**：从「粘贴文本解析」升级到「**文件内容解析**」。

**可能包括**：

- 上传 PDF / Word / Markdown / TXT  
- 从文件中提取正文  
- 再调用 AI 做结构化总结  
- 可能支持更长文本  
- 结果继续保存到项目  

**范围纪律**：**不建议提前过度扩大** — 文件上传涉及文件大小、格式解析、失败状态、隐私提示、存储策略等复杂问题。

**移动端方向（Web H5 基础响应式）**：

- **不做**独立 App；**不做**小程序；**不改变** Web 产品主线。  
- **桌面端优先**，移动端基础可用。  
- 基础适配可放在 **1.4 收尾阶段**：首页输入、解析中、结果页阅读、项目列表、笔记详情、侧栏不溢出、按钮不溢出、无横向滚动。

**1.4 完成后**：Read2Action Web 端基础功能可以 **阶段性收工**。

**1.5 / 2.0**：暂不展开，后续根据真实使用情况再决定。
