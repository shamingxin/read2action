## 1. 版本定位

Read2Action 1.2 在 **1.1 真实 AI 文本总结版** 已冻结的基础上，补全 **解析记录的暂存、归档、找回与空状态体验**，让用户「解析过就能找回」，并区分 **全局未归档** 与 **项目内已归档** 两条路径。

本阶段重点是：

1. 默认真实空状态，mock 不再污染主路径；
2. 全局解析自动 **temporary** 暂存，刷新不丢；
3. 项目内解析自动 **saved** 归档；
4. 保存到项目、原文对照、解析反馈等体验闭环；
5. **不**借机引入数据库、登录、链接抓取或多模型 Provider。

---

## 2. 版本目标（PRD 口径）

1. 真实空状态：首页 / 项目页 / 最近列表无数据时表现正常；
2. 全局自动暂存：不保存到项目也能在最近列表找回；
3. 保存到项目：`/result` 与未归档详情页均可升级 **temporary → saved**；
4. 项目内解析自动归档到当前项目；
5. 原文对照展示用户粘贴的真实输入；
6. 解析中页面单次 POST、真实已用时间反馈；
7. 保持 `/parsing` **A+B+C** 与 `/api/analyze` 稳定链路。

## 5. 关键决策摘要

1. **未归档详情页隐藏面包屑**：避免 `/_temporary` 项目链接触发 **404**；已归档笔记面包屑与项目名链接正常。
2. **`/parsing` 时间展示**：改为 **真实已用时间**（`已用 N 秒`），超 20 秒附加「仍在处理中…」；**不做** AI 耗时预测。
3. **宽屏视觉**：结果页 / 详情页主容器 `r2aContentPageShell`（`max-w-[960px]`）；**不做**整体视觉重构。
4. **原文对照**：贯通 `rawContent` 自 API → session → localStorage → `/result` / 详情页；旧 session 无 `rawContent` 时走拼接兜底。

---

## 6. 已知非阻塞问题

1. **`/result` 无 session 直访**：仍回退 `mockParseResult` 作演示摘要；正常解析主路径不受影响。
2. **保存弹窗项目列表**：含不可创建的「新项目」占位条目（来自 `mockProjects`）。
3. **侧栏「最近」**：最多展示 **4** 条（`.slice(0, 4)`）；PRD 曾建议 10 条，留待后续体验专项。
4. **侧栏「项目」目录**：仍来自预设 `mockProjects`（导航结构，非假笔记数据）。
5. **旧 session / 旧 localStorage 记录**：可能无完整 `rawContent`，原文 Tab 显示兜底拼接而非粘贴全文。
