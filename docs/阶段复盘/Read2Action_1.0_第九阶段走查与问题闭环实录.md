## 2. 视觉与工程：Token 化（9.2 复盘）

### 2.1 做法

将 **页面壳、卡片圆角/阴影、顶栏、Tab 下划线** 等收敛到 **`src/lib/r2a-ui-classes.ts`**（历史文件名，以仓库为准），避免每页复制 Tailwind 长串。

### 2.2 收益

- **跨页一致性**：`/result`、笔记详情、`/parsing`、项目列表、首页纳入同一 **1020** 体系。  
- **后续细修成本下降**：改 token 即可批量影响多页。

### 2.3 代价与注意

- **非像素级一致**仍可能存在（字体、内部 max-w 等），需 **P2/P3** 队列管理，避免在 9.3 无限放大范围。

## 4. 已知「稿差 / 占位」清单（方法论：如何不把它当事故）

以下项来自接力文档 **§7**，属 **预期内技术债** 而非「上线失败」：

- 字体与稿偏差；间距/圆角/阴影非处处像素级。  
- 日期格式不统一（列表 vs 详情）。  
- 内部 `max-w` 与 1020 壳并存。  
- **方法论沉淀** UI 文案仍为占位。  
- **`/result` 原文对照 Tab** 仍为占位。  
- **取消解析确认弹窗** 可能与早期交互稿不一致。

**处理方法**：在 **`docs/核心记录/03_下一步任务清单.md`** 建 **体验专项** 或排入 **1.1 之后**，并标注 **不阻塞 1.0 / 不默认进 1.1 首包**。
