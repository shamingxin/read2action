### 二、本阶段 P0 修复记录（摘要）

**原始 P0**：**`/parsing` 重复请求 `POST /api/analyze`** → **DeepSeek 费用风险**。

**中间问题（排障过程实录）**：

1. 曾出现 **0 条 POST**：**`pendingAnalyzeText`** 被 **提前清理**；**`/parsing` 一闪而过回首页**；Network 过滤 **analyze** 后为 **0** 条 **`POST`**。  
2. 曾出现 **多条 POST**：费用风险 **仍在**。  
3. **A+B+C** 等修复后，重复请求 **疑似压住**，但又出现 **1 条 POST 长期 pending**：页面卡在 **`/parsing`**；Network 显示 **Status unknown / request not finished**（易误判为上游）。  
4. 最终发现本地 **dev server** 出现 **Turbopack FATAL**（`FATAL: An unexpected Turbopack error occurred`）；终端反复 **`GET /parsing` 200**；**干扰**对 **`/api/analyze`** 复验的判断。

**最终修复方案**：

1. **保留 A+B+C 防重复请求**（不推翻）：**`setTimeout(0)`** 替代 **`queueMicrotask`**；**`autoBootRanRef`**；**sessionStorage** 级 **request lock**；**`runId` + `attemptId`**；成功路径 **`writeLastAnalyzeResult` → `clearPending` → `clearAnalyzeRunSessionLocks` → `push("/result")`**。  
2. **`package.json`**：**`"dev": "next dev --webpack"`**（Next **16** 下无 flag 的 **`next dev` 默认 Turbopack**，显式 **webpack** 作本地复验基线）。  
3. **杀掉旧 dev 进程**；**`rm -rf .next`**；**重新 `npm run dev`**。  
4. 确认本地服务：**Next.js 16.2.6（webpack）**、**`http://localhost:3000`**。

- **Read2Action 1.1** 定义为 **「真实 AI 文本总结版」**：在 **不改变 1.0 六页主流程与信息架构** 的前提下，把解析结果从 **mock** 升级为 **由真实大模型基于用户输入文本生成的结构化总结**，并在 **`/result`（及后续保存链路）** 展示 **真实生成结果**，使产品从「可演示原型」迈向「真实可用的最小闭环」。

### 8.2 1.1 只做（范围内）

- 用户 **输入一段待解析的文本**（以 **纯文本** 为 **1.1 唯一可靠输入** 口径）。  
- **调用 AI**（经后端 API，如 `/api/analyze` 或等价路径）生成 **结构化总结**（与现有结果页字段可映射）。  
- **结果页展示真实 AI 输出**（替换当前 mock 解析结果的核心体验）。  
- 与 **`/parsing` 四步加载** 的衔接方式在实现时另定，但 **不扩大 1.1 产品范围** 来换取体验。

### 8.3 1.1 暂不做（范围外）

以下 **明确排除在 1.1 之外**（除非未来 **另立项** 并改写边界）：

- **链接全文抓取 / 自动拉取网页正文**（含任意 URL 解析、爬虫、站内站外全文抓取）。  
- **登录注册**、**账号体系**、**多用户**、**权限与协作**。  
- **数据库** 持久化（项目/笔记/用户数据等 **服务端落库**）。  
- **文件上传** 与基于上传文件的解析管线。  
- **浏览器插件**、RSS/剪藏类 **站外自动采集**。  
- **复杂知识库**（图谱、聊天式知识库、多笔记联合推理等 **超出单段文本总结** 的形态）。  
- **方法论自动生成**（不将「方法论沉淀」在 1.1 内升级为 AI 自动生成模块；与「占位改文案」类 **产品文案** 工作区分）。

### 8.4 用户输入为链接时的产品行为（1.1）

- **1.1 不实现**「粘贴链接 → 自动读取链接正文」。  
- 若判定用户输入 **主要为 URL / 链接形态**，产品侧只做 **轻量提示**（Toast 或输入区旁说明均可，具体 UI 立项后再定），文案口径建议：  
  - **「当前版本暂不支持自动读取链接正文，请复制正文后粘贴。」**  
- 目的：**避免用户误以为 1.1 已具备抓取能力**，从而 **控制预期、收窄工程与合规风险**。

### 8.5 为何现在写死这条边界

- **防止 1.1 范围失控**：真实 AI 一旦接入，最容易被「顺便做抓取、顺便做账号、顺便落库」拖成 **小 2.0**。  
- **先把价值走通**：优先完成 **「文本进 → 真 AI 结构化出 → 结果页真展示」**，再考虑链接、文件、账号与数据层。
