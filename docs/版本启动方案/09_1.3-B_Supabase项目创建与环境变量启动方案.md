## §2 Supabase Dashboard 建项目步骤（人工操作清单）

> **说明**：以下步骤由开发者在浏览器中手动完成，Cursor 不自动执行。完成后将 URL 和 key 填入 `.env.local`。

### Step 1：注册 / 登录 Supabase

- 访问 [https://supabase.com](https://supabase.com)，使用 GitHub 账号或邮箱注册 / 登录。

### Step 2：新建项目

1. 点击「New Project」。
2. **Organization**：选择个人账号（或新建 Organization）。
3. **Project name**：建议 `read2action`（或 `read2action-prod`）。
4. **Database Password**：生成强密码并**妥善保存**（后续可从 Settings 重置）。
5. **Region**：选择距目标用户最近的区域（国内建议 `ap-southeast-1` 新加坡，或 `ap-northeast-1` 东京）。
6. **Plan**：Free tier 足够 1.3 开发与初期使用。
7. 点击「Create new project」，等待约 1～2 分钟完成初始化。

### Step 3：获取环境变量（必须记录以下三项）

进入 **Project Settings → API**：

| 变量名 | 来源 | 说明 |
|--------|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | `Project URL` | 格式 `https://<project-ref>.supabase.co` |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `anon public` key | 可公开，配合 RLS |
| `SUPABASE_SERVICE_ROLE_KEY` | `service_role` key | **仅服务端**，绝不前端 |

> ⚠️ `service_role` key 拥有绕过 RLS 的权限，**严禁**写入 `NEXT_PUBLIC_` 变量或前端代码。

### Step 4：配置 Email Auth 设置

进入 **Authentication → Providers → Email**：

- **Enable Email provider**：✅ 开启
- **Confirm email**（邮箱验证）：
  - **Production**：建议开启（防垃圾注册）
  - **本地 / Preview 开发阶段**：**可暂时关闭**，便于验收；1.3-F 上线前务必开启
- **Secure email change**：默认即可

### Step 5：（可选）配置额外安全设置

进入 **Authentication → URL Configuration**：

- **Site URL**：填写 `https://read2action.vercel.app`（Production）
- **Redirect URLs**：填写 `http://localhost:3000/**`（本地开发用）

### Step 6：执行 SQL migration（1.3-B 代码开工时）

进入 **SQL Editor**，按 §4 草案依次执行。执行顺序：

1. `profiles` 表 + Trigger
2. `projects` 表
3. `notes` 表
4. RLS policies

> **本阶段**：SQL 仅为草案，**不执行**。1.3-C 开工前由开发者在 SQL Editor 中执行。

### Step 7：记录 migration 版本

执行后在 `docs/核心记录/04_重要决策日志.md` 记录执行时间与 commit 关联，作为数据库版本追踪依据。

### 4.1 `profiles` 表

```sql
-- =============================================
-- Table: public.profiles
-- =============================================
CREATE TABLE IF NOT EXISTS public.profiles (
  id                     uuid        PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email                  text,                              -- 冗余存储，方便应用层快速读取；真值来自 auth.users
  display_name           text,
  avatar_url             text,
  migration_status       text        NOT NULL DEFAULT 'none', -- 【1.3-E 预留，1.3-B/C 不实现迁移逻辑】
  migration_completed_at timestamptz,                        -- 【1.3-E 预留】迁移完成时间
  created_at             timestamptz NOT NULL DEFAULT now(),
  updated_at             timestamptz NOT NULL DEFAULT now(),

  CONSTRAINT profiles_migration_status_check
    CHECK (migration_status IN ('none', 'pending_user_confirm', 'in_progress', 'completed', 'failed'))
);

COMMENT ON TABLE public.profiles IS '用户资料表，1:1 对应 auth.users';
COMMENT ON COLUMN public.profiles.email IS 'email 冗余列，来自 auth.users.email；方便应用层直接查 profiles，无需 join auth.users';
COMMENT ON COLUMN public.profiles.migration_status IS '【1.3-E 预留，1.3-B/C 不实现迁移业务逻辑】localStorage 迁移状态：none | pending_user_confirm | in_progress | completed | failed';
COMMENT ON COLUMN public.profiles.migration_completed_at IS '【1.3-E 预留，1.3-B/C 不实现迁移业务逻辑】迁移成功时间';
```

### 4.2 `profiles` 自动创建 Trigger

注册时 `auth.users` 新增一行，自动为该用户创建 `profiles` 行，无需客户端手动调用。

```sql
-- =============================================
-- Function: handle_new_user
-- =============================================
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS trigger
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  INSERT INTO public.profiles (id, email, display_name, migration_status)
  VALUES (
    NEW.id,
    NEW.email,
    COALESCE(NEW.raw_user_meta_data->>'display_name', split_part(NEW.email, '@', 1)),
    'none'
  )
  ON CONFLICT (id) DO NOTHING;
  RETURN NEW;
END;
$$;

-- =============================================
-- Trigger: on_auth_user_created
-- =============================================
DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users;
CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

### 4.3 `projects` 表

```sql
-- =============================================
-- Table: public.projects
-- =============================================
CREATE TABLE IF NOT EXISTS public.projects (
  id          uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     uuid        NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name        text        NOT NULL,
  description text,
  local_id    text,                           -- 原 localStorage projectId（如 'sha'、'_temporary'）
  created_at  timestamptz NOT NULL DEFAULT now(),
  updated_at  timestamptz NOT NULL DEFAULT now()
);

-- 同用户下本地项目只映射一行（幂等迁移核心）
CREATE UNIQUE INDEX IF NOT EXISTS projects_user_local_id_unique
  ON public.projects (user_id, local_id)
  WHERE local_id IS NOT NULL;

-- 列表排序索引
CREATE INDEX IF NOT EXISTS projects_user_updated_at_idx
  ON public.projects (user_id, updated_at DESC);

COMMENT ON TABLE public.projects IS '用户项目表';
COMMENT ON COLUMN public.projects.local_id IS '原 localStorage projectId，用于幂等迁移映射';
```

### 4.4 `notes` 表

```sql
-- =============================================
-- Table: public.notes
-- =============================================
CREATE TABLE IF NOT EXISTS public.notes (
  id              uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         uuid        NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  -- project_id 允许为空：全局解析、最近记录、未归档/暂存记录不一定属于某个项目
  -- 用户保存到项目时再写入；登出 / 迁移场景下项目被删时置 NULL 而非级联删笔记
  project_id      uuid        REFERENCES public.projects(id) ON DELETE SET NULL,
  title           text        NOT NULL,
  source_type     text        NOT NULL DEFAULT 'text',
  source_name     text,
  source_url      text,
  raw_content     text        NOT NULL DEFAULT '',   -- 详情加载，列表查询不 SELECT 此列
  -- 以下结构化字段基于当前 Read2Action 解析结果结构（ParseResultPreview）
  -- 若后续解析结构变化，优先写入 result_json，无需改表；结构化列作为加速查询的冗余
  summary         text        NOT NULL DEFAULT '',
  key_insights    jsonb       NOT NULL DEFAULT '[]',
  action_items    jsonb       NOT NULL DEFAULT '[]',
  knowledge_cards jsonb       NOT NULL DEFAULT '[]',
  result_json     jsonb,                             -- 兜底字段：完整解析结果快照（ParseResultPreview 全量 JSON）
  tags            jsonb       NOT NULL DEFAULT '[]',
  word_count      integer     NOT NULL DEFAULT 0,
  saved_status    text        NOT NULL DEFAULT 'saved',
  source_context  text        NOT NULL DEFAULT 'global',
  local_id        text,                               -- 原 localStorage note.id（防重复迁移核心）
  created_at      timestamptz NOT NULL DEFAULT now(),
  updated_at      timestamptz NOT NULL DEFAULT now(),

  CONSTRAINT notes_saved_status_check
    CHECK (saved_status IN ('temporary', 'saved')),
  CONSTRAINT notes_source_context_check
    CHECK (source_context IN ('global', 'project'))
);

-- 防重复迁移核心约束
CREATE UNIQUE INDEX IF NOT EXISTS notes_user_local_id_unique
  ON public.notes (user_id, local_id)
  WHERE local_id IS NOT NULL;

-- 项目内列表排序索引（project_id 可为 NULL，NULL 行不会命中项目列表查询，无需特殊处理）
CREATE INDEX IF NOT EXISTS notes_user_project_updated_at_idx
  ON public.notes (user_id, project_id, updated_at DESC);

-- 全局最近记录索引（含 project_id IS NULL 的未归档笔记）
CREATE INDEX IF NOT EXISTS notes_user_updated_at_idx
  ON public.notes (user_id, updated_at DESC);

COMMENT ON TABLE public.notes IS '笔记全量字段表（含解析结果）';
COMMENT ON COLUMN public.notes.project_id IS '所属项目；允许为 NULL（未归档/全局暂存）；项目删除时置 NULL 而非级联删笔记';
COMMENT ON COLUMN public.notes.raw_content IS '原文内容，大字段；列表查询应省略此列';
COMMENT ON COLUMN public.notes.result_json IS '完整解析结果 JSON 兜底字段（ParseResultPreview 全量快照）；结构化列（summary/key_insights 等）基于当前解析结构，若后续变化优先补此列';
COMMENT ON COLUMN public.notes.local_id IS '原 localStorage note.id，幂等迁移键；新建云端笔记此列为 NULL';
```

## §5 RLS Policies 草案（文档草案，未执行）

> **原则**：所有业务表默认拒绝；仅通过 Policy 显式放行；`auth.uid()` 为 null（未登录）时等价全部拒绝。

### 5.1 启用 RLS

```sql
-- 三表全部启用 RLS（必须）
ALTER TABLE public.profiles  ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.projects  ENABLE ROW LEVEL SECURITY;
ALTER TABLE public.notes     ENABLE ROW LEVEL SECURITY;
```

### 5.2 `profiles` RLS

```sql
-- SELECT: 只能读自己的 profile
DROP POLICY IF EXISTS "profiles_select_own" ON public.profiles;
CREATE POLICY "profiles_select_own"
  ON public.profiles
  FOR SELECT
  USING (auth.uid() = id);

-- UPDATE: 只能改自己的 profile（display_name、avatar_url 等）
DROP POLICY IF EXISTS "profiles_update_own" ON public.profiles;
CREATE POLICY "profiles_update_own"
  ON public.profiles
  FOR UPDATE
  USING (auth.uid() = id)
  WITH CHECK (auth.uid() = id);

-- INSERT: Trigger 以 service_role 插入；此 policy 允许用户自己 insert（备用）
-- 主路径靠 Trigger，一般不需此 policy；保留供调试
DROP POLICY IF EXISTS "profiles_insert_own" ON public.profiles;
CREATE POLICY "profiles_insert_own"
  ON public.profiles
  FOR INSERT
  WITH CHECK (auth.uid() = id);
```

### 5.4 `notes` RLS

```sql
-- SELECT: 只读自己的笔记
DROP POLICY IF EXISTS "notes_select_own" ON public.notes;
CREATE POLICY "notes_select_own"
  ON public.notes
  FOR SELECT
  USING (user_id = auth.uid());

-- INSERT: 只能为自己创建笔记；若 project_id 非空，该项目必须属于当前用户（防止挂到他人项目）
DROP POLICY IF EXISTS "notes_insert_own" ON public.notes;
CREATE POLICY "notes_insert_own"
  ON public.notes
  FOR INSERT
  WITH CHECK (
    user_id = auth.uid()
    AND (
      project_id IS NULL
      OR EXISTS (
        SELECT 1 FROM public.projects
        WHERE id = project_id
          AND user_id = auth.uid()
      )
    )
  );

-- UPDATE: 只能改自己的笔记；若 project_id 非空，该项目必须属于当前用户（防止改写时挂到他人项目）
DROP POLICY IF EXISTS "notes_update_own" ON public.notes;
CREATE POLICY "notes_update_own"
  ON public.notes
  FOR UPDATE
  USING (user_id = auth.uid())
  WITH CHECK (
    user_id = auth.uid()
    AND (
      project_id IS NULL
      OR EXISTS (
        SELECT 1 FROM public.projects
        WHERE id = project_id
          AND user_id = auth.uid()
      )
    )
  );
```

## §7 Vercel 环境变量配置清单

> **操作路径**：Vercel Dashboard → 选项目 → Settings → Environment Variables

### 7.1 配置步骤

1. 打开 [https://vercel.com](https://vercel.com) → 选择 `read2action` 项目
2. 进入 **Settings → Environment Variables**
3. 按下表逐条添加（每条选择适用的 Environment）

### 7.2 变量配置表

| 变量名 | 值 | Environment | 说明 |
|--------|-----|-------------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | `https://<ref>.supabase.co` | Production + Preview + Development | 可前端 |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `<anon-key>` | Production + Preview + Development | 可前端 |
| `SUPABASE_SERVICE_ROLE_KEY` | `<service-role-key>` | Production + Preview | **仅服务端** |
| `AI_API_KEY` | `<deepseek-key>` | Production + Preview | 与 1.1/1.2 相同 |
| `AI_BASE_URL` | `https://api.deepseek.com` | Production + Preview | 可选 |
| `AI_MODEL` | `deepseek-chat` | Production + Preview | 可选 |

### 7.3 注意事项

- `SUPABASE_SERVICE_ROLE_KEY` **不要**勾选 Development 环境（本地用 `.env.local` 管理）。
- 添加后需 **Redeploy** 才能生效（Settings → Deployments → Redeploy last commit）。
- **Preview 环境**建议指向同一 Supabase 项目（1.3 初期）；1.3-F 上线前可考虑建独立 Preview 项目隔离。
- Vercel 变量添加后，可用 `vercel env pull .env.local` 同步到本地（可选，需 Vercel CLI）。
