## 走查清单补充：Sidebar 项目区（在原有 9.3 清单基础上增加）

以下两项在 **阶段 A（全局）** 或主链路前后插入执行；与「侧栏 Logo / 新笔记 / 进入沙」等检查相邻。

### S1. 真实项目的项目级「⋯」更多菜单

| # | 检查项 | 预期 |
|---|--------|------|
| S1-1 | **入口可见性** | Sidebar **项目列表**中，**真实项目**（来自 mock 的项目行，不含「新增项目」入口，见 S2）在 **hover** 或 **active（当前路由对应项目）** 时，可出现项目级 **「⋯」**（或等价「更多」）入口 |
| S1-2 | **Dropdown** | 点击「⋯」后 **Dropdown** 能正常展开与收起，不闪退、不被裁切 |
| S1-3 | **反馈** | 1.0 **不要求**真实分享 / 重命名 / 删除；菜单内选项点击后至少应有 **Toast** 或 **其它明确反馈**（禁用态说明亦可，但需可感知） |
| S1-4 | **Console** | 操作过程中浏览器 **Console** 无未处理 **error** |

### S2. 「新增项目」不得等同普通 mock 项目

| # | 检查项 | 预期 |
|---|--------|------|
| S2-1 | **区隔** | 「**新增项目**」表现为 **项目区的功能入口**（如独立行、按钮样式或文案区隔），**不应**与 `mockProjects` 中普通项目行完全无差别、仅作为另一条项目链接 |
| S2-2 | **点击行为** | 点击「新增项目」→ **Toast**：「**新建项目功能暂未开放**」（或等价明确占位文案）；**不做**真实新建项目、不写入新 mock 项目 |
| S2-3 | **数据边界** | 确认「新增项目」**不会**被误当作 `projectId` 可进入的 `/projects/[id]` 目标（若错误存在路由，记 **P0/P1**） |
